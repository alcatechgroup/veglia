01 · O QUE É

Um único lugar. Três tipos de conteúdo.

"Documentos, código e decisões na mesma raiz. Quando o dev abre o repo, ele vê o roadmap. Quando o CEO abre a pasta, ele vê o código. Mesmo cérebro, mesma mão."

O command center é a pasta veglia-platform/ — futuramente o repositório GitHub veglia/veglia-platform. Dentro dele convivem três camadas que normalmente ficam separadas: documentos estratégicos (.html, .pdf, .docx), conteúdo educacional (.docx) e código de produção (.tsx, .ts). A vantagem: todo membro do time tem acesso ao contexto inteiro a qualquer momento — sem perguntar onde está o último brandbook ou qual era a decisão sobre o Firebase.

04 · SLASH COMMANDS

Atalhos para ações recorrentes

Cinco comandos custom já configurados em .claude/commands/. Use no Claude Code com / no início do prompt.

/sprint-status
Relatório do progresso do sprint atual nas três frentes (CEO, Dev, Comunicação). Lê CLAUDE.md, verifica git log e estado do código, compara planejado vs realizado.
/briefing-equipe [agente]
Gera briefing semanal por agente do xquads (po, estrategista, dev, designer, edu, copy, etc.) ou para todos. Inclui entregas, dependências e critério de pronto.
/content-semana
Calendário editorial completo da semana para LinkedIn (Vegl.ia + Rodolfo) e Instagram. Posts prontos para colar com headline, corpo, CTA, hashtags.
/decisao-ceo [decisão]
Registra nova decisão estratégica diretamente no CLAUDE.md. Movimenta de "pendente" para "tomada" e analisa o que destrava + risco oculto.
/feature-spec F##
Abre o spec de uma das 22 funcionalidades (F01 a F22) com camada, fase, monetização, KPIs e status atual no projeto.

06 · CONECTAR REPO LOCAL → GITHUB → FIREBASE

Próximos comandos para você rodar

Você já criou o repo GitHub, projeto Firebase e Firestore. Agora vinculamos a estrutura local do command center a tudo isso. Comandos abaixo, em ordem.

Passo 1 · Inicializar Git localmente

# Entrar na pasta do command center cd veglia-platform # Inicializar git git init git branch -M main # Criar .gitignore inicial echo "node_modules/ .env .env.local dist/ .firebase/ .DS_Store .turbo/" > .gitignore # Primeiro commit com toda a base estratégica git add . git commit -m "chore: initial command center with docs, content, and brand assets"

Passo 2 · Conectar ao GitHub

# Vincular ao repo remoto git remote add origin git@github.com:veglia/veglia-platform.git # Push inicial git push -u origin main

Passo 3 · Configurar Firebase CLI

# Login no Firebase (abre browser) firebase login # Vincular pasta local ao projeto Firebase firebase use --add veglia # Inicializar features (escolher: Firestore, Functions, Hosting, Storage) firebase init # Resposta a perguntas: # Use existing project: veglia # Firestore: rules em firestore.rules / indexes em firestore.indexes.json # Functions: TypeScript / install dependencies / Node 20 # Hosting: public = apps/web/dist · SPA: yes · auto build with GitHub: yes # Storage: rules em storage.rules

Passo 4 · Iniciar app web (Vite + React)

# Criar app na pasta apps/web npm create vite@latest apps/web -- --template react-ts cd apps/web npm install # Adicionar Firebase + libs essenciais npm install firebase react-router-dom @tanstack/react-query npm install -D tailwindcss postcss autoprefixer @types/node # Setup Tailwind npx tailwindcss init -p # Setup shadcn/ui npx shadcn-ui@latest init # Voltar para raiz cd ../.. # Primeiro commit do app git add apps/web git commit -m "feat(web): bootstrap React + Vite + TypeScript app" git push

Passo 5 · Configurar GitHub Actions (CI/CD)

# Criar pasta de workflows mkdir -p .github/workflows # GitHub Actions de auto-deploy ao Firebase Hosting # (script firebase deploy --only hosting + auth via service account) # Pedir ao Claude Code: @dev configurar GitHub Actions para deploy automático no Firebase Hosting (staging em PR, prod em merge para main)

Commit das mudanças. Se houve decisão estratégica, rodar /decisao-ceo para registrá-la. Push para o GitHub.
