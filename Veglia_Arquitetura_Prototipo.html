03 · ESTRATÉGIA DE VÍDEO

Canal privado YouTube com player embedded

A decisão de hospedar vídeos no YouTube (e não no Firebase Storage) é estratégica. Vamos extrair o melhor dos dois mundos: infraestrutura de streaming gratuita do YouTube + experiência blindada dentro da Vegl.ia.

Por que YouTube e não Firebase Storage?

CritérioYouTube (Não listado)Firebase Storage
Custo de bandaZero (Google paga)R$ 0,50/GB transferido
Streaming adaptativoNativo (HLS/DASH)Precisa Cloud CDN + transcoding
Player com trackingIFrame API completaImplementação manual
Backup de mídiaPermanente (Google)Custo do backup
Tempo de carregamentoOtimizado globalmenteDepende da região

Veredito: em escala B2B com milhões de visualizações de colaboradores, a economia de banda do YouTube é a diferença entre operar com margem e operar no zero. O canal "Não Listado" oferece privacidade prática sem custo de hosting.

Configuração do canal

Setup técnico

  • → Canal: Vegl.ia · Conteúdo Educacional Corporativo
  • → Visibilidade: Não Listado (acesso só por link, mas embeddable)
  • → Player params: rel=0 (sem vídeos relacionados), modestbranding=1 (sem logo YouTube), controls=1, fs=1
  • → Domínios autorizados via header Referrer-Policy + verificação no IFrame
  • → Captions/legendas habilitadas em PT-BR (acessibilidade WCAG 2.1 AA)

Tracking de progresso

Via YouTube IFrame Player API, capturamos eventos onStateChange (PLAYING / PAUSED / ENDED) e poll a cada 5s para capturar getCurrentTime(). Salvamos no Firestore por user_id × video_id: posição atual, % assistido, primeiro acesso, último acesso, completou. Vídeo só conta como concluído após 90% assistido sem pular trechos.

Experiência embedded

O player roda dentro de um componente React custom da Vegl.ia. O usuário nunca sai da plataforma — não vê interface YouTube fora do player, não vê vídeos sugeridos, não pode compartilhar o link. Visualmente parece nativo. Tecnicamente é IFrame API gerenciada por nós.

Modelo de dados (Firestore)

Estrutura inicial das collections principais — pensada para isolamento multi-tenant via company_id em cada documento e Security Rules robustas.

companies/{companyId}
  ├─ name, cnpj, plan, white_label_partner
  ├─ modules_active: [moduleId]
  └─ branding: { logo, primary_color, domain }

users/{uid}
  ├─ company_id, role, percurso
  ├─ name, email, cargo, departamento
  └─ stats: { modules_completed, hours_total }

modules/{moduleId}
  ├─ track: "lei_15377" | "nr1"
  ├─ percurso: "colaborador" | "gestor"
  ├─ videos: [{ youtube_id, duration, order }]
  └─ quiz: [{ question, options, correct }]

enrollments/{enrollmentId}
  ├─ user_id, module_id, company_id
  ├─ progress_pct, status
  └─ started_at, completed_at

video_progress/{user_id}_{video_id}
  ├─ watched_seconds, completion_pct
  ├─ first_watched_at, last_watched_at
  └─ completed: boolean

certificates/{certId}
  ├─ user_id, module_id, company_id
  ├─ sha256_hash, pdf_storage_path
  └─ issued_at, valid_until

Segurança & Compliance

Firestore Security Rules garantem que cada usuário só acessa documentos do seu company_id via custom claims. Admins de carriers (VR, Alelo) têm visibilidade agregada de todas as empresas que contrataram via eles. Master admin Vegl.ia tem visão global. Todas as operações sensíveis (acesso a certificado, conclusão de módulo) geram entrada em audit_logs imutável — base para auditoria trabalhista da Lei 15.377.

06 · DECISÕES A TOMAR ESTA SEMANA

Quatro escolhas que destravam tudo

Antes de escrever a primeira linha de código, precisamos resolver isto. Algumas têm recomendação clara, outras dependem do alinhamento com Fábio e Thiago.

Decisão 01 · Domínio e ambiente
Vegl.ia ou veglia.com.br como domínio principal?
Opções: (A) vegl.ia nativo · (B) veglia.com.br tradicional · (C) ambos com redirect. Recomendado: opção C — registrar os dois, usar app.vegl.ia como produto e veglia.com.br como institucional/marketing.
Decisão 02 · Plano Firebase
Spark (gratuito) ou Blaze (pay-as-you-go) já no protótipo?
Recomendado: Blaze. Cloud Functions exige Blaze, geração de PDF idem. Custo estimado nos primeiros 60 dias: < R$ 200/mês. Vale a previsibilidade.
Decisão 03 · Quem escreve o código
Sócios codificam, contratamos dev externo ou ambos?
Rodolfo lidera Tech Lead + arquitetura. Recomendação: 1 dev React/Firebase contratado (PJ ou vesting parcial) para acelerar Fase 1. Fábio cuida de operação comercial. Thiago opera relação Vacivitta. Sócios não devem ser gargalo de execução técnica.
Decisão 04 · Gravação dos vídeos
Quando começar a produção dos 12 vídeos do roteiro?
Recomendado: gravar 1 vídeo piloto nas próximas 2 semanas (módulo Vacinação Corporativa) com Dra. Amanda e protótipo de cenário. Validar formato, identidade visual em vídeo, qualidade de áudio. Só depois escalar produção dos 11 restantes.
07 · DIAGNÓSTICO ESTRATÉGICO CEO

O que está realmente em jogo

Diagnóstico

Vegl.ia tem janela curta. A Lei 15.377/2026 entra em vigor com prazo de adequação corporativa, e o primeiro player que oferecer compliance preventivo embalado em white label B2B2B captura o mercado. A vantagem competitiva está na combinação rara: ecossistema Vacivitta (10 anos de operação), autoridade médica (Dra. Amanda), tecnologia ágil (RRevela/Rodolfo) e canal de distribuição (Fábio com benefits carriers). Nenhum concorrente tem isso junto.

Análise de Mercado

TAM: ~5 milhões de empresas no Brasil com obrigação de compliance trabalhista; SAM imediato via VR ~80 mil empresas-clientes. Janela competitiva: 12-18 meses antes de incumbents legais (Conjur, ContaAzul) ou edtechs (Alura, Rocketseat) perceberem a oportunidade vertical e entrarem.

Decisão Recomendada

🟢 AVANÇAR — agora. Stack Firebase + GitHub + YouTube embedded é a escolha certa: minimiza tempo até MVP (30 dias), preserva caixa, escala sem reescrita. O próximo movimento é setup técnico e contratação do dev React/Firebase. Não criar primeiro um app mobile — web responsivo (PWA) cobre 95% dos casos B2B nessa fase.

Risco Oculto

O risco real não é técnico — é cadência de produção de conteúdo. 6 módulos × 2 vídeos cada = 12 produções audiovisuais com Dra. Amanda. Se não houver disciplina semanal de gravação, a plataforma fica linda mas vazia. Solução: contratar um produtor audiovisual freelance dedicado por 60 dias, não fazer tudo internamente.

A Pergunta que Você Deveria Estar Fazendo

"A Vegl.ia precisa ser a melhor edtech B2B do Brasil — ou só precisa ser a primeira boa o suficiente que VR aceitar como white label?"

A resposta muda completamente a régua de qualidade do MVP. Se for "a primeira boa o suficiente", entregamos em 30 dias e iteramos com cliente real. Se for "a melhor", atrasamos 4 meses para fazer mais bonito — e perdemos a janela.
