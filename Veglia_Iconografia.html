— I — Princípios visuais

3 regras que fazem um ícone parecer da Vegl.ia.

Sistema autoral nasce de regras claras. Cada ícone do Linecraft respeita 3 princípios não-negociáveis que diferenciam a marca de qualquer biblioteca genérica (Lucide, Feather, Material Symbols).

01 · Linha única, peso constante

Stroke 1.5px em viewport 24×24. Sem variação de espessura. Sem fill maciço. A linha é o protagonista.

02 · Geometria circular como ancoragem

O círculo é a base de quase todos os ícones — alinhamento direto ao ponto do logotipo Vegl·ia. Reforça coesão visual.

03 · Ponto champagne como assinatura

Em ícones-chave (vigília, certificação, validação), o ponto dourado marca presença. Discreto. Reservado a momentos de autoridade.

— II — Anatomia do ícone

Construção em grade matemática.

Cada ícone é desenhado dentro de uma grade 24×24 com área útil 20×20 (padding 2px de respiro). Stroke linecap e linejoin sempre round. Cantos com radius 1-2 quando aplicável.

Padding 2px

Área útil de 20×20 dentro do viewport 24×24. Garante respiro em qualquer aplicação.

Centralização absoluta

Eixos central e horizontal sempre como referência. Simetria onde possível, intencionalidade onde quebrada.

Stroke round

linecap="round" + linejoin="round" sempre. Pontas arredondadas evitam dureza visual.

Acento champagne

Ponto preenchido em #C9A96E como assinatura em ícones especiais. Nunca decorativo — sempre semântico.

— XIII — Acessibilidade

WCAG 2.1 AA desde o primeiro pixel.

Regras inegociáveis para todo ícone Vegl.ia

1. Sempre acompanhado de aria-label descritivo. O leitor de tela precisa entender a função, não a forma. Ex: aria-label="Adequação à Lei 15.377", não aria-label="escudo".

2. Ícone sozinho exige role="img". Se for clicável, role="button" + aria-label adicional.

3. Cor sozinha nunca comunica estado. Erro tem cor + ícone de alerta + texto. Concluído tem cor + check + texto.

4. Contraste mínimo 4.5:1. Em qualquer plano de fundo da paleta, o stroke do ícone precisa passar no teste de contraste WCAG AA.

5. Foco visível obrigatório. Quando ícone é interativo, anel de foco em mint (#5DD3A8) com 2px de offset.

6. Não-decorativo precisa de title. SVG inline com <title> é parseado pelo leitor de tela mesmo sem aria-label.

— XIV — Handoff técnico · @dev

Como o time técnico consome o sistema.

Estrutura de entrega

Cada ícone vive em um arquivo SVG individual em src/icons/vi-[nome].svg. O componente React/Vue/Web Component recebe props padronizadas:

<Icon
  name="vi-compliance"
  size={24}
  color="currentColor"
  ariaLabel="Adequação à Lei 15.377"
  hasGoldDot={true}
/>

Convenções

ItemConvenção
Namingvi-[função-em-kebab-case] · ex: vi-compliance, vi-vaccine, vi-team
CorcurrentColor sempre · controlado via CSS no contexto
Ponto champagneAtributo fill="#C9A96E" stroke="none" · usado só em ícones-manifesto e validação
Tree-shakingCada ícone é módulo isolado · bundle paga só pelos ícones usados
Sprite SVGVersão sprite disponível para uso via <use href="#vi-name"> em landing
VersionamentoSemVer · MAJOR ao adicionar/remover ícones · MINOR ao melhorar visual sem quebrar API

Próximos passos para o @dev

1. Exportar SVGs individuais a partir deste documento (todos estão inline aqui — fácil extração).

2. Configurar bundler (Vite, Webpack) com loader SVG-as-component.

3. Criar wrapper <Icon> com props padronizadas e lógica de aria-label.

4. Documentar no Storybook (ou similar) para o time consumir.

5. Versionar pacote @veglia/icons internamente para reuso entre landing, app web, app mobile.
